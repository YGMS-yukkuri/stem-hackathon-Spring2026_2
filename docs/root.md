# 仕様書の構成
- 仕様書は、docs/ディレクトリの下にMarkdown形式で記述されています。
- 各機能や要件は、個別のMarkdownファイルに記述されており、トップレベルのドキュメントからリンクされています。
- 仕様書は、プロジェクトの全体像を把握するためのガイドラインや、各機能の詳細な説明を提供しています。

# 仕様書の目的
- 仕様書は、プロジェクトの実装に必要な要件や機能を明確にするためのドキュメントです。
- 仕様書を参照しながら、実装を進めることで、プロジェクトの品質や精度を向上させることができます。
- 仕様書に記載されていない要件や機能については、質問し、ユーザーの返答を仕様書に追加することで、より明確な仕様書づくりや、実装の精度向上に努めてください。

# 仕様書の更新
